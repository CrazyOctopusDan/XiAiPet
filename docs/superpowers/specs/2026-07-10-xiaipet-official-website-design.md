# XiAiPet 宠物烘焙官网设计

**日期：** 2026-07-10
**状态：** 待用户审阅
**范围：** `apps/official-website/` 的单页品牌官网。首轮不改动现有小程序、API 或交易流程。

## 目标

让第一次到访的宠物家长在较短时间内确认三件事：

1. 食物值得放心给宠物吃。
2. XiAiPet 理解生日、纪念日和日常陪伴的庆祝需求。
3. 可以立即前往现有小程序选购和下单。

官网负责建立信任与情感，不重复建设商品交易、会员和订单管理。

## 已确认的方向

选择 A「新鲜守护仪式」。视觉和文案同时表达“安心”与“仪式感”，避免粉色卡通宠物店和传统米色手作烘焙模板。

### 品牌基调

- 继承客户小程序的真实基础色：`#F9F1C7`、`#FFFBEA`、`#40535C` 和 `#D9A15B`。官网将其扩展为更有留白和影像层次的桌面品牌界面，而不是直接放大成小程序样式。
- 暖金 `#D9A15B` 是唯一行动与庆祝强调色；石板墨色 `#40535C` 负责文字、导航和轮廓，保持与小程序视觉连续。
- 食物摄影是叙事主体。页面不以抽象渐变或插画代替真实烘焙产品。
- `Noto Serif SC` 仅用于少量大标题，`Noto Sans SC` 承担主体信息，避免过度装饰。
- 轻量 Canvas 食材粒子作为首屏背景增强。指针靠近时粒子会柔和避让，点击时短暂聚拢为庆祝感的光点；它不承载内容，在关闭动画或低性能设备上退化为静态构图。

## 页面结构

1. **首屏**：一句明确主张、两项行动入口和一张真实的成品蛋糕近景。Canvas 粒子会跟随指针产生轻微避让与聚拢，表达“认真准备”的过程。
2. **放心从看得见开始**：用配料、现制与适配需求三种信息呈现信任依据。这里使用不等尺寸的图文单元，不做整齐的六宫格。
3. **每个重要日子，都值得认真准备**：以生日、纪念日、日常奖励三个真实场景组织内容，用一张横向可滑动的照片叙事而不是虚构评价。
4. **从确认到分享**：用真实的三步服务流程说明咨询、确认需求、预约领取或配送。编号仅用于这个真实顺序。
5. **正在制作的作品**：从现有商品目录加载真实的已发布商品缩略图。图片墙支持悬停聚焦、键盘聚焦与移动端横向拖动；选择商品仅展示名称与简介，并引导至小程序，不在官网实现购物车。
6. **最终行动区**：引导打开小程序浏览。二维码、客服、门店与配送信息仅在取得真实资料后展示。
7. **页脚**：保留品牌、服务范围、社交渠道和必要备案信息的位置。

## 交互与可访问性

- 导航在滚动后使用半透明深色底，不使用泛滥的玻璃卡片。
- 动效只服务于首屏层次、图片进入和流程理解；全站最多一个横向滚动叙事区。
- 每个链接与按钮拥有可见焦点样式，移动端触控区域不小于 44px。
- 「安心配方 / 生日庆祝 / 日常奖励」使用可访问的内容切换控件：切换文案与相邻照片构图，不伪造筛选结果或评价。
- 商品图片墙从接口加载后提供骨架屏、空状态和失败提示；图片不可用时保留产品名称与小程序行动入口。
- 首屏、图片和核心行动在不执行 Canvas 动画时仍完整可读；`prefers-reduced-motion` 下停用动画。
- 正文、按钮和提示文案按 WCAG AA 对比度校验；不使用纯白、纯黑或硬边深灰。

## 内容与资料边界

- 不编造营养数据、配方认证、用户评价、地址、营业时间或价格。
- 商品图从 `GET /api/v1/customer/catalog/products` 的已发布商品摘要读取，使用其 `thumbnail` 字段。该接口无需客户登录，适合成为官网产品图来源。
- 不调用 `GET /api/v1/customer/orders`。该接口需要客户会话，并会返回个人订单快照，不应成为公开官网的素材源。
- 若还需要首屏或品牌故事照片，再列出用途与素材清单并取得单次明确授权。
- 小程序入口、二维码、门店地图、客服和备案信息等待真实资料接入。

## 技术边界

- 官网独立存放于 `apps/official-website/`，作为本地开发和服务器部署的唯一代码目录。
- 第一版默认是静态品牌站，和既有小程序/API 通过链接或二维码衔接，不新增官网侧交易接口。
- 生产环境通过同源反向代理把官网的 `/api/v1/customer/catalog/products` 转发到现有 API。当前 API 源码未见跨域白名单配置，因此不以浏览器跨域直连作为默认方案。
- 实现开始前再确定前端脚手架及部署方式，避免在未确认服务器环境时引入不必要的运行时依赖。

## 验收标准

- 访客可以在移动端清楚理解“安全配方”和“庆祝时刻”两条价值主张。
- 不出现默认宠物卡通、米色黄铜烘焙模板、紫色发光、渐变文字、伪造数字或假评价。
- 首屏存在可降级的视觉引擎，并且全部核心内容在禁用动效时可访问。
- 真实素材、业务信息与小程序入口均有明确来源，不以占位数据伪装上线信息。
