# Miniapp Regression Checklist

This checklist is the Phase 12 manual regression source for the customer and merchant mini programs after the HTTP API and OSS migration. Run the automated references first, then use designated test users, test orders and non-production payment paths for manual checks.

## Customer Mini Program Regression

| Workflow | Automated reference | Manual steps | Expected result | Status |
| --- | --- | --- | --- | --- |
| launch/login/bootstrap | `pnpm --filter @xiaipet/customer-miniapp test` covers customer auth login bootstrap through page tests and `apps/customer-miniapp/src/services/api-client.ts`. | Open the customer mini program in WeChat DevTools with the API base URL pointed at the intended test API. Launch from a cold start and allow `wx.login` to complete. | A customer session token is stored, bootstrap requests use `/api/v1/customer/*`, and no `wx.cloud.callFunction` backend call is needed. | Pending manual regression |
| home/runtime config banner | `pnpm --filter @xiaipet/customer-miniapp test` plus runtime config request handling in `apps/customer-miniapp/pages/cart-checkout.test.ts`. | Visit the home page and refresh runtime config. Check store copy, delivery rules and Banner display against the configured test data. | Runtime config renders from the HTTP API; missing optional Banner or store values degrade cleanly without blocking navigation. | Pending manual regression |
| catalog categories/products | `pnpm --filter @xiaipet/customer-miniapp test` exercises catalog/cart fixtures and API-backed page behavior. | Open catalog tabs, switch categories, inspect published products and member/stock labels. | Category and product cards load from the migrated API data with correct price, stock, member visibility and fulfillment labels. | Pending manual regression |
| search/detail | `pnpm --filter @xiaipet/customer-miniapp test` covers product selection paths used by cart and checkout tests. | Search by a known product keyword, open the product detail page, switch available specs or formulas and return to catalog. | Search results and detail content match API data; unavailable products or specs cannot be selected. | Pending manual regression |
| cart and spec selection | `apps/customer-miniapp/pages/cart-checkout.test.ts` and `pnpm --filter @xiaipet/customer-miniapp test`. | Add a direct product and a spec/formula product to the cart, change quantities, clear one item, then re-open the cart. | Cart totals, selected quantity, spec labels and line prices update deterministically and survive page navigation as expected. | Pending manual regression |
| checkout fulfillment/address/pet/remark | `apps/customer-miniapp/pages/cart-checkout.test.ts` and order creation request assertions. | Start checkout with selected cart items. Choose delivery, pickup and express where available; select a saved address, pet and reservation time; enter a remark. | Checkout snapshot includes fulfillment mode, address or pickup store, reservation, pet, remark, delivery fee and payable total before submit. | Pending manual regression |
| payment start using available mock/dev path | `apps/customer-miniapp/pages/cart-checkout.test.ts` covers `POST /api/v1/customer/orders` and `POST /api/v1/customer/orders/{id}/payment`. | Submit a test order using the available mock/dev payment path only. Do not use real WeChat Pay until production credentials and callback verification are ready. | Order enters pending payment, mock/dev payment start returns a controlled response, and the UI does not claim real payment acceptance. | Pending manual regression |
| payment sync | `apps/customer-miniapp/pages/cart-checkout.test.ts` covers `/api/v1/customer/orders/{id}/payment-sync`. | Trigger payment status refresh for the test order after the mock/dev payment path completes. | The customer mini program calls payment sync, shows the latest server-side order payment status and avoids client-side status fabrication. | Pending manual regression |
| order list/detail | `apps/customer-miniapp/pages/orders-flow.test.ts` and `pnpm --filter @xiaipet/customer-miniapp test`. | Open the order list, inspect empty and populated states, then enter a test order detail. | Order cards and detail show status, payment, fulfillment, snapshot items, pet, remark and total from the HTTP API. | Pending manual regression |
| profile | `pnpm --filter @xiaipet/customer-miniapp test` covers customer session retry and profile phone update request handling. | Open profile after login, verify nickname/avatar display if present, and bind or refresh masked phone with a test account. | Profile data reads from the API and phone masking/update behavior does not expose raw credentials or secrets. | Pending manual regression |
| pets | `pnpm --filter @xiaipet/customer-miniapp test` covers page-level customer API behavior used by profile workflows. | Add, edit and delete a test pet from the customer pet management pages. | Pet list updates through authenticated customer API calls and only the test account's pet data changes. | Pending manual regression |
| address list/form | `pnpm --filter @xiaipet/customer-miniapp test` covers checkout address selection dependencies. | Add a test address, edit recipient/phone/region/detail, set it as selectable for checkout, then remove it if needed. | Address list and checkout selector reflect the same server data; invalid phone or missing address fields are blocked before submit. | Pending manual regression |
| OSS-backed image display | `pnpm --filter @xiaipet/customer-miniapp test` plus OSS asset migration checks from Phase 11. | Inspect product covers, detail images and runtime Banner using migrated OSS URLs such as `https://xiaipet-assets-prod.oss-cn-hangzhou.aliyuncs.com/...`. | Images render from OSS-backed URLs accepted by WeChat legal-domain configuration after ICP approval; no CloudBase storage URL is required for production display. | Pending manual regression |

## Merchant Mini Program Regression

Use designated merchant test accounts and test orders only. Do not run destructive balance or order-state checks against real customers or production fulfillment orders unless the cutover owner explicitly approves the exact record IDs.

| Workflow | Automated reference | Manual steps | Expected result | Status |
| --- | --- | --- | --- | --- |
| access gate login and whitelist rejection | `pnpm --filter @xiaipet/merchant-miniapp test` covers merchant auth client behavior and protected request handling. | Login with a whitelisted merchant test account, then repeat with a non-whitelisted test account. | Whitelisted merchant enters the dashboard through `/api/v1/merchant/auth/login`; non-whitelisted login is rejected and no protected merchant data is displayed. | Pending manual regression |
| order list/detail | `apps/merchant-miniapp/src/services/orders.test.ts` and `pnpm --filter @xiaipet/merchant-miniapp test`. | Open the merchant order list using seeded test orders, filter or group by status, then enter a test order detail. | Order groups, detail snapshot, payment badge, fulfillment state, timeline and audit summary match server data. | Pending manual regression |
| order status update | `apps/merchant-miniapp/src/services/orders.test.ts` covers `updateMerchantOrderStatus`. | On a designated test order, move fulfillment status through the allowed next state and record the operator. | Status update succeeds only through the merchant API, timeline/audit metadata is added, and unrelated orders are unchanged. | Pending manual regression |
| catalog category management | `apps/merchant-miniapp/src/services/catalog-admin.test.ts` covers category query, save and delete API calls. | Create or edit a test category, verify linked product count copy, and attempt delete only on a disposable category. | Category changes persist through `/api/v1/merchant/categories`; categories with linked products require migration before delete. | Pending manual regression |
| product create/update including OSS upload | `apps/merchant-miniapp/src/services/catalog-admin.test.ts` and `apps/merchant-miniapp/src/services/assets.test.ts`. | Create or edit a test product, upload a cover image through the merchant image picker, and save pricing/spec/formula settings. | Product save preserves OSS `storageId` and public preview URL; upload uses merchant upload policy and confirmation endpoints without exposing OSS AK/SK. | Pending manual regression |
| user search | `apps/merchant-miniapp/src/services/user-admin.test.ts` covers explicit submit payloads for user search. | Search for designated test users by phone/nickname/openid where supported. | Search runs only after explicit submit, returns masked user details and never exposes raw credentials or full payment secrets. | Pending manual regression |
| balance adjustment safeguard with test user only | `apps/merchant-miniapp/src/services/user-admin.test.ts` covers negative-balance blocking and confirmation payloads. | Select a designated balance test user, preview add/deduct/set-target adjustments, confirm one safe mutation and record the before/after balance. | Negative resulting balance is blocked; confirmed adjustments include operator identity, before balance, delta/target, after balance and reason. | Pending manual regression |
| runtime config/Banner editing | `pnpm --filter @xiaipet/merchant-miniapp test` plus merchant runtime config API regression. | Edit store copy, delivery rules or test Banner content, then refresh the customer home page in a test environment. | Merchant edits persist through the API and customer runtime config displays the updated safe content without CloudBase fallback. | Pending manual regression |
| printer settings | `pnpm --filter @xiaipet/merchant-miniapp test` plus receipt service checks in order tests. | Open printer settings with a test printer profile, change device label or mode if available, and save. | Printer settings are stored for receipt preparation without attempting an unintended physical print. | Pending manual regression |
| receipt print preparation | `apps/merchant-miniapp/src/services/orders.test.ts` covers receipt metadata and printable order conditions. | On a paid designated test order, open receipt preview or preparation flow before printing. | Receipt data includes order items, totals, fulfillment info, customer masked contact, template version and printer target. | Pending manual regression |
| receipt print result recording | `apps/merchant-miniapp/src/services/orders.test.ts` covers receipt print count/result metadata on order detail. | Record a test print success and a controlled failure if the UI supports it, using only a designated test order. | Print count, last print time, result, device label and audit timeline are updated through the merchant API. | Pending manual regression |
