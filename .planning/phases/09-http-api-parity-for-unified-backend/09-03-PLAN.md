<objective>
Expose customer order creation, payment start, payment sync/confirmation, order list, and order detail APIs under `/api/v1/customer`.
</objective>

<threat_model>
T9-03A: A customer creates or pays an order for another user by passing another `openid`.
Severity: high.
Mitigation: All order service methods accept `openid` from session guard only and ignore body/query ownership fields.

T9-03B: Mock WeChat payment output is treated as production payment settlement.
Severity: high.
Mitigation: `PaymentProvider` clearly separates mock/dev behavior from production; real callback signature verification is documented as Phase 12.
</threat_model>
