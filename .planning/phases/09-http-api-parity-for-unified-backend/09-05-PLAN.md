<objective>
Expose merchant admin APIs for categories, products, user search, balance adjustment, and runtime configuration.
</objective>

<threat_model>
T9-05A: Non-whitelisted users access admin catalog, user, balance or runtime config data.
Severity: high.
Mitigation: Every route uses `requireMerchantSession`; tests assert fake services are not called on denial.

T9-05B: Balance adjustment payloads bypass validation and create unaudited money changes.
Severity: high.
Mitigation: Route delegates to balance service using shared adjustment schema and writes ledger/audit data transactionally.
</threat_model>
