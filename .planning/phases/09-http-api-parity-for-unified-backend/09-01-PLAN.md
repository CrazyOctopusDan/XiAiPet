<objective>
Create the Phase 09 HTTP authentication foundation and expose parity APIs for `bootstrapUser`, `bindPhone`, and `assertMerchantAccess`.
</objective>

<threat_model>
T9-01: A client spoofs another user by sending `openid` in the request body or query string.
Severity: high.
Mitigation: Protected routes read identity only from a verified signed session token populated by `requireCustomerSession` or `requireMerchantSession`.

T9-02: A non-whitelisted user calls merchant APIs and route handlers fetch sensitive data before authorization.
Severity: high.
Mitigation: `requireMerchantSession` verifies `merchant_users` before handler execution; route tests assert fake sensitive services are not called on denial.
</threat_model>
