<objective>
Migrate customer checkout submission, payment start/sync and order list/detail services from CloudBase calls to authenticated HTTP API requests.
</objective>

<threat_model>
T10-02A: Checkout sends payment or ownership fields that override server-side authority.
Severity: high.
Mitigation: Keep `buildCreateOrderPayload()` focused on order content only; rely on bearer session for user identity; tests assert no `openid` is sent.

T10-02B: Payment failures expose backend internals or break existing page states.
Severity: medium.
Mitigation: Map API errors such as `INSUFFICIENT_BALANCE`, `ORDER_NOT_FOUND`, `REQUEST_FAILED`, and `REQUEST_TIMEOUT` to stable service errors consumed by current page tests.
</threat_model>
