<objective>
Migrate merchant admin services for catalog, users/balances, runtime config and receipt printing from CloudBase functions to the Phase 9 merchant HTTP API.
</objective>

<threat_model>
T10-04A: Merchant balance adjustment or runtime config update executes without backend merchant authorization.
Severity: high.
Mitigation: All admin mutations use `merchantApiRequest` with bearer token and Phase 9 merchant guards; local operator data is metadata only.

T10-04B: Product image upload accidentally keeps CloudBase as hidden backend dependency for migrated operations.
Severity: medium.
Mitigation: Keep image upload outside migrated HTTP operation paths and mark it Phase 11/OSS pending if it cannot be migrated now.
</threat_model>
