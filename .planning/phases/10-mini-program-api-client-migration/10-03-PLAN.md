<objective>
Add the merchant miniapp HTTP API client, then migrate merchant access and order operations away from `wx.cloud.Cloud` / CloudBase function calls.
</objective>

<threat_model>
T10-03A: A non-merchant user reaches merchant data because the miniapp trusts local state instead of backend authorization.
Severity: high.
Mitigation: `verifyMerchantAccess()` and all merchant order calls use bearer session and backend `requireMerchantSession`; handle `MERCHANT_FORBIDDEN` as denied access.

T10-03B: Existing dirty merchant service files are overwritten during migration.
Severity: medium.
Mitigation: Executor must read current merchant files before editing and preserve unrelated user changes.
</threat_model>
