<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| merchant miniapp -> Cloud Functions | Untrusted operator input enters category and product CRUD handlers |
| Cloud Functions -> categories/products collections | Admin writes can break referential integrity or persistence rules if validation is weak |
| merchant upload UI -> CloudBase storage metadata | Uploaded asset identifiers can be tampered with or persisted as expiring URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | E | category/product CRUD handlers | mitigate | Require merchant access on every catalog handler and keep `categories`/`products` backend-only in security rules. |
| T-06-02-02 | T | category delete + product save logic | mitigate | Enforce D-18 referential checks, single-category validation, required fulfillment modes, and shared pricing validators before any write. |
| T-06-02-03 | I | product image and banner-style asset handling | mitigate | Persist CloudBase `fileID` values only and derive preview URLs at read time instead of storing temp URLs. |
</threat_model>
