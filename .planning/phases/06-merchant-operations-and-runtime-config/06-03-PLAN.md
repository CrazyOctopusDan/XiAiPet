<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| merchant miniapp -> user/config Cloud Functions | Untrusted admin input enters privileged user lookup, balance, and config handlers |
| Cloud Functions -> balance accounts/ledgers | Financial writes cross the transaction boundary and must stay auditable |
| Cloud Functions -> runtime config -> customer storefront | Admin-entered config becomes customer-visible data and map/banner behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | I/E | `searchMerchantUsers` | mitigate | Keep the query behind merchant auth, mask phones in the response model, and return only D-19 fields. |
| T-06-03-02 | T/R | `adjustUserBalance` and `payment-store` | mitigate | Use a server transaction to update account plus ledger together, reject negative target balances, and record operator/reason/note/before/after balances for every adjustment. |
| T-06-03-03 | T/I | `upsertRuntimeConfigSection` and customer config readers | mitigate | Persist fixed-key section docs with per-section metadata, store banner `fileID` rather than temp URLs, and validate required fields before any customer surface consumes them. |
</threat_model>
