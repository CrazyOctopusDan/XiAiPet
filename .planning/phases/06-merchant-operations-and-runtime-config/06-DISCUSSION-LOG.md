## 订单状态流转

| Option | Description | Selected |
|--------|-------------|----------|
| 订单列表分组 | A. 按履约进度分组；B. 按支付结果分组；C. 先看待商户处理再看其他历史状态 | A |
| 已支付订单推进 | A. 三段轻量状态；B. 按履约方式细分中间态；C. 只做已处理/已完成 | B |
| 未支付订单处理 | A. 只查看；B. 可手动取消；C. 可手动标记为已支付/已处理 | C |
| 回退规则 | A. 只允许单向推进；B. 允许有限回退；C. 允许任意切换 | C |

**User's choice:** 列表按履约进度分组；已支付订单按不同履约方式细分状态；未支付订单允许商户人工标记为已支付/已处理；非终态之间允许灵活切换。
**Notes:** 追问后补充锁定了三条履约状态链：`delivery=待处理>制作中>配送中>已完成`、`pickup=待处理>制作中>待自取>已完成`、`express=待处理>制作中>待发货>已完成`。人工兜底必须记录操作人、时间、前后状态、原因备注和调整方式。终态锁死。客户端直接展示同一套细分状态文案。

## 品类与商品编辑模型

| Option | Description | Selected |
|--------|-------------|----------|
| 商户端信息架构 | A. 五个独立入口；B. 四个入口，品类并入商品管理；C. 首页管理卡片 | C |
| 商品编辑形态 | A. 单页长表单；B. 分区式单页；C. 先列表，再进详情后分步骤编辑 | C |
| 品类与商品关系 | A. 商品只挂一个一级品类；B. 商品可挂多个品类；C. 弱关联展示 | A |
| 规格/配方/价格录入 | A. 配方主导，规格附加；B. 规格和配方独立行项组合；C. 仅基准价 + 配方加价 | B |

**User's choice:** 商户端首页走管理卡片入口；商品首版只挂一个一级品类；商品通过列表进入详情并分步骤编辑；规格和配方独立建模。
**Notes:** 追问后补充锁定了编辑步骤 `基础信息 → 规格配方与价格 → 上架设置`。组合价格默认自动相加，但允许少数组合手动覆盖最终价。删除一级品类前必须先迁移商品。商品列表默认按品类浏览。

## 用户搜索与余额调整规则

| Option | Description | Selected |
|--------|-------------|----------|
| 搜索结果展示 | A. 轻量列表后进详情操作；B. 结果内直接改余额；C. 先极简字段再异步加载 | A |
| 余额调整动作 | A. 只增加；B. 增加和扣减；C. 增加、扣减、直接指定余额 | C |
| 记录深度 | A. 仅自由文本；B. 原因类型 + 可选备注；C. 完整落账 | C |
| 风险控制 | A. 提交前统一二次确认；B. 超额才确认；C. 不需要确认 | A |

**User's choice:** 搜索页先轻量展示，余额支持增加、扣减和直接改到指定值，每次都必须完整落账，提交前统一二次确认。
**Notes:** 追问后补充：指定余额是日常同级动作，不仅限纠错；原因类型采用固定预置枚举 `充值/补偿/人工纠错/线下收款/其他`；用户侧余额流水展示“标题 + 简短备注”；余额不能被直接改成负数。

## 运营配置范围与分组

| Option | Description | Selected |
|--------|-------------|----------|
| 入口组织 | A. 一个运营配置入口，进去后按分区编辑；B. 首页拆多个配置卡片；C. 只做少量配置 UI | A |
| 保存方式 | A. 每个分区单独保存；B. 整页统一保存；C. 混合保存 | A |
| 配送费规则 | A. 按距离阶梯；B. 固定一口价；C. 基础配送费 + 超距加价 | C |
| Banner 管理 | A. 图片 + 跳转；B. 多图排序上下架；C. 单张主 Banner | C |

**User's choice:** 运营配置保持单入口、分区单独保存；配送费按设计稿中的距离与价格阶梯录入；Banner 首版只支持单张主 Banner。
**Notes:** 追问后补充：会员等级阈值需要支持门槛、等级名称和说明文案；定制提示采用单条长文本 + 开关启用；店铺信息只开放地址、地图坐标和联系电话。用户明确引用 `req/img/配送费提示.png` 作为配送费配置的 canonical 设计来源，并提供了完整阶梯文案。
