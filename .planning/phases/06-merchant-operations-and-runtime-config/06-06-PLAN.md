## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| merchant miniapp -> user/runtime Cloud Functions | Untrusted admin lookup, balance, and config payloads reach privileged handlers |
| Cloud Functions -> balance accounts/ledgers and runtime configs | Financial and customer-visible writes cross backend-controlled storage boundaries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06-01 | I/E | `searchMerchantUsers` | mitigate | Enforce merchant auth and return only whitelisted D-19 fields with masked phone output. |
| T-06-06-02 | T/R/I | `adjustUserBalance` / `payment-store` | mitigate | Use a transaction for account plus ledger writes, require full audit metadata, reject negative resulting balances, and persist separate D-25 `normalizedTitle`/`shortNote` fields so customer readers never need the full internal note. |
| T-06-06-03 | T/I | `upsertRuntimeConfigSection` | mitigate | Validate fixed-key section payloads, store banner `fileID` instead of temp URLs, and isolate each save to one section document. |
| T-06-06-04 | I | `readRuntimeConfig` | mitigate | Return only the storefront-safe fixed-key projection and keep merchant admin handlers (`getRuntimeConfigSections`, `upsertRuntimeConfigSection`) behind merchant auth. |
</threat_model>
