<objective>
Create the backend and shared-type foundation for OSS assets: validated config, signed upload primitives, shared role/variant asset contracts, and product persistence fields for cover/introduction/detail image metadata.
</objective>

<threat_model>
T11-01A: Long-lived OSS credentials leak through config, logs, upload-policy responses, or tests.
Severity: high.
Mitigation: `OSS_ACCESS_KEY_SECRET` remains only in `ApiConfig`; policy response snapshots and logger redaction tests assert the secret string is absent.

T11-01B: A merchant can upload or confirm a file outside their controlled prefix.
Severity: high.
Mitigation: `createOssObjectKey` generates keys only under `merchant/{merchantId}/assets/{role}/YYYY/`, and confirm tokens bind merchantId, role, variant, objectKey, expiresAt, width, height, and size limits.

T11-01C: Frontend later loads full-size images in product lists because metadata does not include role-specific variants.
Severity: medium.
Mitigation: shared asset schema requires variant names and URLs; product-cover must support `thumbnail` and `display` variants.
</threat_model>
