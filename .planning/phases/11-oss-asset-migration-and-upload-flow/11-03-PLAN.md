<objective>
Implement an idempotent report-first migration path for existing `cloud://` references so Phase 11 can identify, skip, or optionally migrate legacy CloudBase assets without blocking new OSS uploads.
</objective>

<threat_model>
T11-03A: Migration script overwrites valid OSS references with fake or missing CloudBase placeholders.
Severity: medium.
Mitigation: Default mode is report-only; update/copy mode requires explicit `--apply` and source map arguments, and tests assert existing `oss://` references are classified as `alreadyOss`.

T11-03B: Report accidentally includes secrets or local file contents.
Severity: medium.
Mitigation: Report only includes entity id, field path, reference value, classification, target key, and reason; it does not embed file bytes or credentials.
</threat_model>

      - `findAssetReferences(input)` scans products, runtime config sections, and arbitrary seed records for string values starting `cloud://` or `oss://`.
      - `classifyAssetReference(reference, sourceMap)` returns one of `migratable`, `placeholder`, `missingSource`, `alreadyOss`, `skipped`.

      - `cloud://` without source map entry => `missingSource`
      - empty or local `/assets/` values => `skipped`
