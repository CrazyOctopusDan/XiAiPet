<objective>
Wire OSS asset references into customer display and merchant runtime-config editing, then document the operational requirements for OSS bucket/CORS/domain setup and image-cost controls.
</objective>

<threat_model>
T11-04A: Customer pages load oversized detail images for list/card or home banner surfaces.
Severity: medium.
Mitigation: Display helpers prefer `thumbnail` for product list/cards and `banner` for home banner; tests assert selected URLs.

T11-04B: Runtime banner upload bypasses merchant authorization or stores raw local paths.
Severity: high.
Mitigation: Banner upload reuses `uploadMerchantAsset('runtime-banner')`; saved runtime config stores `oss://` id plus HTTPS URL asset metadata, not `wxfile://` paths.

T11-04C: Operator misconfigures OSS and WeChat domains, causing production upload/display failure.
Severity: medium.
Mitigation: Release docs list required OSS bucket policy, CORS methods/headers, public URL behavior, and WeChat legal upload/download domain configuration.
</threat_model>
