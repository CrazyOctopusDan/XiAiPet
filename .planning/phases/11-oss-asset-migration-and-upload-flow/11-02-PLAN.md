<objective>
Expose protected merchant asset upload endpoints and implement the merchant miniapp service/page flow for role-driven image selection, crop/compression, OSS direct upload, and API confirmation.
</objective>

<threat_model>
T11-02A: Unauthorized user obtains an OSS upload policy and stores public objects.
Severity: high.
Mitigation: Both asset routes use `dependencies.guards.requireMerchantSession`; denied merchant tests assert service is not called.

T11-02B: Miniapp uploads a different file/key than the backend authorized.
Severity: high.
Mitigation: Upload policy includes exact object key, and confirm request must include the backend `confirmToken`; service rejects mismatched role/key/variant.

T11-02C: Oversized images increase OSS traffic and miniapp load times.
Severity: medium.
Mitigation: `assets.ts` applies role rules before upload and rejects files above target size after compression.
</threat_model>
